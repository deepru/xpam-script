# XPAM Script v1.3.7 — примечания к релизу

Дата: 2026-07-01
Базовая версия: v1.3.6

Релиз посвящён чистке архитектуры (удаление устаревших профиля и MTProto-бэкенда),
новому восстановлению базы 3x-ui через `repair --full`, проверке нехватки памяти,
полной совместимости с Debian 13 / Ubuntu 26.04 и инфраструктуре сборки релиза.

> Совместимость 3x-ui/Xray: без изменений сопряжения. Последняя проверенная база —
> 3x-ui 3.4.2 / Xray 26.6.27 (upgrade и fresh-install).

> **Совместимость ОС:** релиз полностью протестирован и совместим с **Debian 13** и
> **Ubuntu 26.04** (оба профиля, чистая установка + repair + repair --full + weekly +
> deep-health — всё зелёное), а также с более ранними Ubuntu/Debian.

---

## Что нового для пользователя

- **Восстановление базы 3x-ui: `sudo <prefix>-repair --full`.**
  Обычный `repair` восстанавливает обвязку сервера из шаблонов и не трогает данные.
  Новый `repair --full` дополнительно **восстанавливает базу 3x-ui** (клиенты,
  inbound'ы, MTProto-секрет) из последнего golden-снапшота:
  - проверка целостности снапшота (`PRAGMA integrity_check`);
  - явное подтверждение (`restore-full`) с предупреждением, что клиенты/секреты,
    созданные после снапшота, будут потеряны;
  - бэкап текущей базы перед восстановлением;
  - **авто-откат**, если проверка health после восстановления не прошла.

- **`repair` теперь также регенерирует конфиг nginx** (раньше — только haproxy),
  то есть чинит больше поломок «обвязки» без переустановки.

- **Health: новая проверка нехватки памяти** (доступная RAM, использование swap,
  события OOM) — на уровне WARN, без ложных тревог на здоровом сервере.

- **Полная совместимость с Debian 13 и Ubuntu 26.04** — протестировано на чистой
  установке (оба профиля, repair, repair --full, weekly, deep-health — всё зелёное).

## Изменения (важно прочитать)

- **Удалён устаревший профиль `vless_direct`.** Сервер всегда работает по схеме
  HAProxy → локальный Xray (это и была схема по умолчанию).

- **Удалён legacy MTProto-бэкенд `alexbers`.** MTProto работает только через
  встроенный в 3x-ui MTG (3xui-mtg).
  - **Команда `<prefix>-tg` удалена.** Telegram-ссылка теперь показывается через
    `sudo <prefix>-links --show-secrets` (вместе с VLESS-ссылкой).
  - Мульти-пользовательское управление MTProto (config.py от alexbers) удалено —
    в модели 3xui-mtg используется одна управляемая ссылка.

- **Fail-fast при импорте старых конфигов.** Конфиг с удалённым профилем/бэкендом
  (`vless_direct`/`alexbers`/`teleproxy`) отклоняется с понятным сообщением —
  такие серверы нужно устанавливать заново на чистой ВМ.

## Безопасность

- Хардненинг сокета Telegram-relay: убран небезопасный fallback на права `0o666`
  (мир-запись) — теперь fail-safe `0o660` root-only с предупреждением.
- Задокументированы намеренные loopback-only отключения проверки TLS
  (обращения к панели 3x-ui по 127.0.0.1).

## Для мейнтейнера / инфраструктура

- **`make-release.sh`** — сборка релизного архива с обязательной top-level
  wrapper-директорией + `.sha256`, и верификация ровно теми же гейтами, что
  применяет self-updater (root_count==1, обязательные файлы, `bash -n`,
  render-smoke, sha256). Защита от класса регрессий упаковки.
- **CI** (`.github/workflows/ci.yml`) — сборка+верификация на каждый push/PR
  плюс advisory `shellcheck`.
- Удаление мёртвого кода после снятия `vless_direct`/`alexbers`.

## Обновление

Существующие серверы обновляются штатным self-update
(`sudo <prefix>-xpam` → обновление). Архив собран с корректной wrapper-структурой,
поэтому staging апдейтера принимает его.
