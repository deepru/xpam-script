# XPAM Script v1.0.8 - короткий обзор

XPAM Script - это автоматизированный набор для подготовки чистого VPS под VLESS, MTProto, 3x-ui, HAProxy, nginx, TLS-сертификаты, SSH-защиту, диагностику и регулярное обслуживание.

Главная задача проекта - не просто "поставить proxy", а собрать понятную и проверяемую HTTPS-поверхность. Снаружи сервер использует только стандартные публичные TCP-порты 22, 80 и 443. VLESS, панель 3x-ui, MTProto, Telegram HTTPS Relay и служебные проверки прячутся за доменами, TLS, SNI-маршрутизацией, локальными backend-портами и статичными сайтами-маскировками.

## Для чего нужен скрипт

XPAM Script полезен, когда пользователь хочет быстро подготовить VPS без ручной сборки десятков конфигов. Скрипт помогает:

- закрыть SSH-пароль и оставить вход только по ключу;
- выпустить и подключить TLS-сертификаты Let’s Encrypt;
- поднять 3x-ui и VLESS с корректными публичными ссылками через домен:443;
- добавить MTProto через отдельный домен без отдельного публичного MTProto-порта;
- сделать нормальные статичные сайты-маскировки для доменов;
- настроить nginx, HAProxy, Xray, Certbot, UFW, fail2ban и systemd-службы;
- включить еженедельное обслуживание и проверку состояния сервера;
- убрать лишние демоны и временный мусор с VPS;
- получить единые команды управления: `<prefix>-install`, `<prefix>-links`, `<prefix>-vless`, `<prefix>-telega`, `<prefix>-health`.

## Что меняется на сервере

Скрипт настраивает SSH, firewall, DNS-политику, TLS, nginx, HAProxy, 3x-ui/Xray, MTProto, certbot hooks, service limits, swap и network tuning. Внешне остаются только нужные публичные порты: 22, 80 и 443. Внутренние сервисы слушают локально и проверяются health-check.

Для облегчения VPS XPAM Script отключает или удаляет службы, которые обычно не нужны для такой роли сервера: snapd, packagekit, fwupd, apport, unattended-upgrades, apt-daily timers, motd-news, update-notifier, open-iscsi, multipathd и другие лишние фоновые компоненты, если они присутствуют на образе.

## Маскировка HTTPS-поверхности

VLESS и MTProto не торчат наружу отдельными подозрительными портами. Домен VLESS открывается как обычный HTTPS-сайт и одновременно служит входной точкой для VLESS и закрытой панели 3x-ui. MTProto-домен в MTProto-профилях выглядит как sync/API endpoint, а реальный MTProto backend находится за HAProxy и локальным портом.

Это не магия и не гарантия абсолютной невидимости. Пользователь всё равно обязан правильно настроить домены, DNS, сертификаты, SSH-ключи и хранение секретов. Но схема делает сервер аккуратным, повторяемым и гораздо менее хаотичным, чем ручная установка из разрозненных команд.

## Требования

Для работы нужен VPS на Ubuntu 24.04 или Debian 12, root-доступ, заранее созданный SSH-ключ и собственный домен с возможностью редактировать DNS-записи. Все используемые домены и поддомены должны указывать A-записями на IPv4-адрес VPS.

## Дополнительные функции

Telegram-уведомления можно отправлять напрямую или через HTTPS Relay-сервер XPAM Script. Уведомления нужны не для шума, а для важных событий: ошибка еженедельной проверки или необходимость ручной перезагрузки после обновления.

WARP реализован не через системный warp-cli, а через WireGuard outbound внутри 3x-ui/Xray. Это позволяет маршрутизировать через WARP только выбранные домены и IP, не меняя default route и DNS всего VPS.

Статичные сайты можно менять: пользователь может загрузить свои страницы в `/var/www/<domain>`, но не должен занимать служебные route Let’s Encrypt, панели 3x-ui, MTProto health/API и Telegram Relay.

## Итог

XPAM Script приводит VPS к понятной роли: защищённый SSH, чистая HTTPS-поверхность, VLESS, MTProto, управляемые сайты, автоматическое обслуживание и глубокий health-check. Он не заменяет понимание администратора, но резко уменьшает количество ручной рутины и риск забыть важный шаг.
